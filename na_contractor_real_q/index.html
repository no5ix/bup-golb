React Questions

  1. What is Virtual DOM in React?
    The Virtual DOM is an in-memory representation of the real DOM. React updates it first, diffs it, and only patches the changed nodes to the actual DOM.

  2. What are React Hooks (useState, useEffect, useMemo)?
    Hooks let function components use state and lifecycle logic. useState manages local state, useEffect handles side effects, and useMemo caches computed values.

  3. What is useEffect used for?
    useEffect runs after render and handles side effects like API calls, subscriptions, and cleanup logic. It runs on mount, update, and unmount depending on the dependency array.

  4. What is a controlled vs uncontrolled component?
    Controlled components store input value in state, updated via onChange. Uncontrolled ones use refs to access input value directly.

  5. What is prop drilling? How do you avoid it?
    Prop drilling is passing props through multiple intermediate components. It can be avoided using React Context API or global state libraries like Redux.

  6. What is the difference between props and state?
    Props are external, read-only data passed from parent to child. State is internal and mutable, managed within the component.

  7. What is the React component structure and data flow?
    React uses a top-down (unidirectional) data flow. Parent components pass data via props, and child components can lift state updates via callbacks.

  8. What is an Error Boundary?
    An Error Boundary is a class component that catches rendering errors in its child tree. It renders fallback UI instead of crashing the entire app.

  9. What is a Higher Order Component (HOC)?
    A Higher Order Component is a function that takes a component and returns a new component with additional features. It’s used for logic reuse like authentication or data fetching.

  10. Describe the React component lifecycle.
    Functional components use useEffect to simulate lifecycle phases. It runs after mount, on update (with dependencies), and supports cleanup on unmount.

  11. How does Redux work? What are reducer/store/dispatch?
    Redux uses a global store to hold state. dispatch() sends actions, and reducers are pure functions that return the updated state based on those actions.

  12. Difference between Redux and Context API?
    Context API is built into React and is ideal for simple global data like theme or auth. Redux is better for large-scale apps, with middleware, devtools, and structured state management.

Spring Boot Newbie Notes

In Java, specifically in the Spring framework, both @Controller and @RestController are annotations used for
building web - related components. Here are the key differences between them:

1. Function and Purpose

  • @Controller
    • It is a stereotype annotation in Spring MVC. It is mainly used to mark a class as a Spring MVC controller. A
      controller class typically handles HTTP requests, processes them, and then returns a view name. The view resolver
      in Spring MVC will then use this view name to render an appropriate view (such as a JSP page, Thymeleaf template,
      etc.) to the client.
  • @RestController
    • It is a combination of @Controller and @ResponseBody. This annotation is designed for building RESTful web
      services. When a class is marked with @RestController, all the handler methods in this class will automatically
      serialize the return value to the HTTP response body in a format like JSON or XML by default.

  • @Controller
    • By default, the return value of a method in a @Controller class is treated as a view name. For example:

      example, an e - commerce website where you need to display product lists, user profiles, etc. in HTML pages.

Avro with Confluent Schema Registry

Memory

memory improvement

Here are some common methods to improve memory usage in Java:

1. Optimize Object Creation

  • Reuse Objects: Instead of creating new objects repeatedly, reuse existing ones when possible. For example, in a loop where you need to format dates, reuse a SimpleDateFormat object instead of creating a new one each time.

  • Use Object Pools: For objects that are expensive to create, such as database connections or threads, use object pools. Libraries like Apache Commons Pool can help you implement object pools easily.

2. Manage Memory Efficiently in Collections

  • Choose the Right Collection: Select the appropriate collection type based on your usage scenario. For example, if you need a collection with fast look - up, use HashMap; if you need a sorted collection, use TreeMap.

  • Trim Collections: If you are using collections like ArrayList or HashMap, and you know that the size will not increase significantly after a certain point, you can trim the excess capacity. For ArrayList, you can use the trimToSize() method.

3. Avoid Memory Leaks

  • Properly Close Resources: Make sure to close resources such as files, database connections, and network sockets properly. You can use the try - with - resources statement introduced in Java 7 to ensure automatic resource management.

  • Weak References: Use weak references (WeakReference) when you need to hold a reference to an object but don’t want to prevent it from being garbage - collected. This is useful in caching scenarios.

4. Tune Garbage Collection

  • Choose the Right Garbage Collector: Java provides different garbage collectors, such as Serial GC, Parallel GC, CMS (Concurrent Mark Sweep) GC, and G1 (Garbage - First) GC. You can choose the appropriate garbage collector based on your application’s requirements. For example, if your application has low pause time requirements, G1 GC might be a good choice. You can specify the garbage collector using JVM options, like -XX:+UseG1GC.
  • Adjust Heap Size: You can adjust the initial and maximum heap size of the JVM using the -Xms and -Xmx options. For example, -Xms512m -Xmx1024m sets the initial heap size to 512 MB and the maximum heap size to 1024 MB.

what are the scopes in OAuth process

In the OAuth (Open Authorization) process, scopes are used to define the specific permissions that a client application requests from the resource owner. They act as a way to limit the access that the client has to the protected resources of the resource owner. Here are some common aspects and examples of scopes in the OAuth process:

1. General Concept

Scopes are strings that represent a set of related permissions. When a client application initiates the OAuth authorization flow, it includes a list of scopes in the authorization request. The resource owner is then presented with these scopes and can choose to grant or deny the requested permissions. The authorization server issues an access token that is limited to the scopes that the resource owner has approved.

2. Examples of Scopes in Different Services

Social Media Platforms

  • Facebook:
    • email: Allows the client application to access the user’s Facebook - associated email address.
    • user_posts: Grants the application the ability to read the user’s posts on Facebook.
    • publish_pages: Enables the application to publish content on the user’s Facebook pages.
  • Twitter:
    • tweet.read: Permits the application to read the user’s tweets and timeline.
    • tweet.write: Allows the application to post tweets on behalf of the user.
    • users.read: Gives the application access to basic user information such as the user’s profile.

Cloud Storage Services

  • Google Drive:
    • https://www.googleapis.com/auth/drive.readonly: Grants read - only access to the user’s Google Drive files.
    • https://www.googleapis.com/auth/drive: Provides full access to the user’s Google Drive, including read, write, and delete operations.

Email Services

  • Gmail:
    • https://www.googleapis.com/auth/gmail.readonly: Allows the application to read the user’s Gmail messages.
    • https://www.googleapis.com/auth/gmail.send: Enables the application to send emails on behalf of the user.

3. Importance of Scopes

  • Security: Scopes help in enhancing security by ensuring that the client application only has access to the resources and operations that the resource owner explicitly approves. This reduces the risk of a malicious or misconfigured application accessing sensitive information.
  • User Control: They give users more control over their data. Users can review the scopes requested by an application and decide whether to grant the permissions based on their trust in the application.

difference between encryption and hashing

Encryption and hashing are two fundamental concepts in the field of data security, but they serve different purposes and have distinct characteristics. Here’s a detailed comparison between them:

1. Purpose

  • Encryption: Its main purpose is to transform data into an unreadable format so that it can be safely transmitted or stored. The original data, known as plaintext, can be recovered by using a decryption key. This is crucial for protecting sensitive information such as financial transactions, personal data, and confidential communications. For example, when you make an online purchase, your credit card details are encrypted to prevent unauthorized access during transmission.
  • Hashing: Hashing is used to generate a fixed - size string of characters (hash value) from input data. It is mainly employed for data integrity verification and password storage. A hash value can be used to check if the data has been altered, as even a small change in the input data will result in a completely different hash value. Passwords are often hashed so that they can be stored securely without revealing the actual password.

2. Process

  • Encryption: It involves an encryption algorithm and a key. The encryption algorithm takes the plaintext and the key as inputs and produces ciphertext. There are two main types of encryption: symmetric encryption (uses the same key for encryption and decryption, like AES) and asymmetric encryption (uses a pair of keys - a public key for encryption and a private key for decryption, such as RSA).
  • Hashing: A hash function takes the input data and generates a hash value through a one - way process. There is no key involved in the traditional sense. Popular hash functions include MD5, SHA - 1, SHA - 256, and SHA - 512. Once the hash value is generated, it is not possible to reverse - engineer the original input data from the hash.

3. Reversibility

  • Encryption: It is a reversible process. If you have the correct decryption key, you can convert the ciphertext back to the original plaintext. This reversibility is essential for scenarios where the data needs to be accessed in its original form after being encrypted.
  • Hashing: It is a one - way function, which means it is not possible to reverse the process and obtain the original input data from the hash value. The only way to verify if a given input matches the original data is to hash the new input and compare the resulting hash values.

4. Output Size

  • Encryption: The output size (ciphertext) is usually proportional to the size of the input plaintext, with some additional overhead depending on the encryption algorithm. For example, in AES encryption, the output size is the same as the input size for block - based encryption.
  • Hashing: The output size is fixed regardless of the size of the input data. For instance, the output of the SHA - 256 hash function is always 256 bits (32 bytes), whether the input is a single character or a large file.

5. Use Cases

  • Encryption:
    • Secure communication channels (e.g., SSL/TLS for HTTPS)
    • Encrypting data at rest (e.g., encrypting a hard drive or a database)
    • Protecting sensitive files and folders
  • Hashing:
    • Password storage in user accounts
    • Verifying the integrity of downloaded files (e.g., checking the hash of a software download)
    • Blockchain technology for maintaining the integrity of transaction data

5. SQL - group by roll up（SQL 的 GROUP BY ROLLUP）

GROUP BY ROLLUP 是 SQL 中的一个高级分组功能，它能生成额外的汇总行。例如：

此查询会生成按部门和职位分组的汇总数据，还会生成部门级别的汇总数据和全局汇总数据。
SELECT department, job_title, SUM(salary) FROM employees GROUP BY ROLLUP (department, job_title); 这条 SQL 语句运用了 GROUP BY ROLLUP 子句，它是 SQL 里用于高级分组和生成汇总数据的强大功能。下面详细解释此语句的各个部分以及 ROLLUP 的作用。

语句各部分解释

1. SELECT 子句

这部分指定了要从 employees 表中选取的列。

  • department：表示员工所在的部门。
  • job_title：代表员工的职位。
  • SUM(salary)：对 salary 列进行求和操作，也就是计算每个分组内员工工资的总和。

2. FROM 子句

此部分指明了数据的来源表，即从 employees 表中获取数据。

3. GROUP BY ROLLUP 子句

这是该语句的核心部分，ROLLUP 是一种特殊的分组操作，它会在 GROUP BY 子句的基础上生成额外的汇总行。具体来说，ROLLUP (department, job_title) 会按照以下顺序进行分组：

  1. 首先按照 department 和 job_title 进行分组，计算每个部门和职位组合下的工资总和。
  2. 接着按照 department 进行分组，计算每个部门的工资总和。
  3. 最后生成一个全局汇总行，计算所有员工的工资总和。

示例数据与结果

假设 employees 表有如下数据：

department job_title salary
IT Developer 5000
IT Tester 4000
HR Recruiter 3500
HR Trainer 3000

执行 SELECT department, job_title, SUM(salary) FROM employees GROUP BY ROLLUP (department, job_title); 后，结果可能如下：

department job_title SUM(salary)
IT Developer 5000
IT Tester 4000
IT NULL 9000 – IT 部门的汇总
HR Recruiter 3500
HR Trainer 3000
HR NULL 6500 – HR 部门的汇总

总结

在记事本里编码比较基础，适合简单脚本或代码片段的编写。不过它缺乏语法高亮、代码提示等功能，调试和维护代码相对困难。可以考虑使用更专业的代码编辑器，如 Visual Studio Code、PyCharm 等。

7. Multiple questions（多个问题）

处理多个问题时，可按问题的优先级和复杂度逐个解决。先解决关键问题和容易解决的问题，再处理复杂问题。

8. Restful APIs（RESTful API）

RESTful API 是一种遵循 REST 架构风格的 API 设计原则。其特点有：

  • 使用 HTTP 方法：如 GET（获取资源）、POST（创建资源）、PUT（更新资源）、DELETE（删除资源）。
  • 使用 URL 定位资源：例如 /users/1 表示 ID 为 1 的用户。
  • 无状态：每个请求都是独立的，服务器不保存客户端的状态信息。

9. Previous work experience（以往工作经验）

描述以往工作经验时，可从项目背景、担任角色、主要职责、取得成果等方面入手。比如：“在[项目名称]中，我担任[角色]，负责[具体工作内容]，通过[具体方法]，成功[取得的成果]。”

10. Multi threading（多线程）

多线程是指在一个程序里同时执行多个线程。多线程能提高程序的并发性能，尤其适用于 I/O 密集型和 CPU 密集型任务。在 Java 中，可通过继承 Thread 类或实现 Runnable 接口来创建线程。示例代码如下：

11. Microservices（微服务）

微服务是一种将应用拆分成多个小型、自治服务的架构风格。每个服务都能独立开发、部署和扩展，通过轻量级通信机制（如 HTTP、消息队列）进行交互。微服务架构的优点包括提高开发效率、增强系统的可扩展性和容错性等。

  • 复杂度：代码复杂度、理解难度等。
  • 适用性：适用场景、数据规模等。

14. Ask for inputs and outputs for coding questions（询问编码问题的输入输出）

在解决编码问题时，明确输入和输出很重要。输入是程序运行所需的数据，输出是程序运行后的结果。例如，对于判断回文的问题，输入是一个字符串，输出是一个布尔值，表示该字符串是否为回文。

15. Design patterns（设计模式）

设计模式是软件开发中针对反复出现问题总结出的通用解决方案。常见的设计模式有单例模式、工厂模式、观察者模式等。单例模式确保一个类只有一个实例，并提供全局访问点；工厂模式用于创建对象，将对象的创建和使用分离。

16. API gateways（API 网关）

API 网关是一个位于客户端和后端服务之间的中间层，负责接收客户端的请求，对请求进行路由、转换、过滤、认证等处理，然后将请求转发到相应的后端服务，并将后端服务的响应返回给客户端。API 网关可以提高系统的安全性、可维护性和可扩展性。

17. Book - gang of 4 - the person who designed the design patterns（《设计模式：可复用面向对象软件的基础》作者）

《设计模式：可复用面向对象软件的基础》（Design Patterns: Elements of Reusable Object - Oriented Software）通常被称为“四人组（Gang of Four）”的书，作者是 Erich Gamma、Richard Helm、Ralph Johnson 和 John Vlissides。

面对快速提问，要保持冷静，集中注意力，迅速理解问题的核心，然后有条理地给出回答。如果问题复杂，可以先简要说明思路，再详细解释。

26. Backend（后端）

后端开发负责处理业务逻辑、数据存储和管理等。常见的后端技术栈有 Java + Spring Boot、Python + Django/Flask、Node.js + Express 等。

27. Kafka（Kafka）

Kafka 是一个分布式流处理平台，常用于处理高吞吐量的实时数据流。它具有高吞吐量、可扩展性、持久性等特点。Kafka 的主要组件有生产者（Producer）、消费者（Consumer）、主题（Topic）和分区（Partition）。

28. Rabbit MQ（RabbitMQ）

RabbitMQ 是一个开源的消息队列中间件，遵循 AMQP（高级消息队列协议）标准。它可用于实现异步通信、解耦系统组件等。RabbitMQ 的主要组件有生产者、消费者、交换机（Exchange）、队列（Queue）和绑定（Binding）。

29. Dataflow in your project（项目中的数据流）

描述项目中的数据流时，可从数据的来源、处理过程和去向等方面入手。例如，数据从前端表单收集，经过后端服务器的业务逻辑处理，存储到数据库中，然后根据需求从数据库中读取数据并展示到前端页面。

30. Design pattern used in your project（项目中使用的设计模式）

在项目中，可根据具体需求使用不同的设计模式。例如，使用单例模式确保数据库连接池只有一个实例，使用工厂模式创建不同类型的对象。

32. Framework rules（框架规则）

不同框架有不同的规则和使用方式。使用框架时，要熟悉框架的文档和规范，按照框架的规则进行开发。例如，使用 Spring Boot 时，要遵循其自动配置、注解使用等规则。

33. Saga pattern microservices（微服务中的 Saga 模式）

Saga 模式是一种用于处理分布式事务的模式，适用于微服务架构。当一个业务流程涉及多个服务的操作时，Saga 模式通过一系列的本地事务来实现最终一致性。每个本地事务都有一个对应的补偿事务，当某个本地事务失败时，会执行相应的补偿事务来撤销之前的操作。

35. DB indexes（数据库索引）

数据库索引是一种数据结构，用于提高数据库查询的效率。常见的索引类型有 B - 树索引、哈希索引等。通过在经常用于查询条件的列上创建索引，可以减少数据库的扫描范围，加快查询速度。但索引也会增加数据库的存储空间和写操作的开销，因此需要根据实际情况合理创建索引。

36. Springboot annotation（Spring Boot 注解）

Spring Boot 提供了许多注解，用于简化开发。常见的注解有：

  • @SpringBootApplication：用于标记主应用类，开启自动配置、组件扫描等功能。
  • @RestController：用于标记控制器类，将其方法返回值直接作为 HTTP 响应体返回。
  • @GetMapping、@PostMapping 等：用于映射 HTTP 请求方法。
